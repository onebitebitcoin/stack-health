# 프로젝트 비전 — 운동하고 비트코인 받자

> **모든 에이전트는 구현 전 이 문서를 반드시 읽는다.**
> 이 비전은 기능 우선순위, 설계 결정, 리워드 구조의 최상위 기준이다.

---

## 핵심 명제

> **비트코인 리워드는 진입 후크다. 진짜 동기는 커뮤니티와 성장 피드백이다.**

이 앱은 단순 숏츠 플랫폼이 아닌 **운동 성장 일기(Workout Growth Diary)** 다.  
개인은 꾸준히 나아질 때 성취감을 느끼고, 그 경험을 같은 관심사를 가진 사람과 공유하고 싶어한다.  
우리는 그 액션 루프를 만든다.

---

## 액션 루프

```
기록(Record) → 반응(Reaction) → 기록(Record) → ...
```

- 운동 후 15초 영상을 찍는다
- 다른 사람이 격려하고, 질문하고, 따라 한다
- 더 나은 다음 운동이 동기화된다

---

## 무엇을 기록하는가

**영상 형식**: 15초 숏츠  
**콘텐츠 예시**: 운동 장면, 신발, 땀, 장소 — 날 것의 현장감  
**대상 운동**: 달리기, 헬스, 걷기, 홈트레이닝, 요가, 필라테스, 수영, 자전거 — 초기부터 모든 운동 카테고리를 포함하며, 특정 카테고리에 집중하지 않는다 (감독자 결정 2026-05-22)

### 업로드 전 5가지 질문 (Phase B)

업로드 직전, 앱이 묻는다:
1. 오늘 무엇을 했나요?
2. 얼마나 했나요?
3. 가장 힘들었던 점은?
4. 다음에 개선하고 싶은 것은?
5. 다른 사람에게 묻고 싶은 것은?

> **Phase B 구현**: 텍스트 오버레이로 피드에 표시되어 "맥락 있는 기록"을 만든다.

---

## 사용자 역할 (Phase B)

| 역할 | 설명 |
|------|------|
| **일반 운동인** | 운동을 기록하고 커뮤니티와 교류 |
| **어드바이저** | 전문 피드백 제공, 루틴 설계, 커뮤니티 리더십 |

> 어드바이저는 기여도에 따라 추가 리워드를 받는다. 수익화 모델(구독 또는 수수료)은 Phase B 설계 시 확정.

---

## 리워드 설계 원칙

### 핵심 원칙: 외부 수익 연동 지속 가능 모델

StepN과 같은 신규 유입 폰지 구조를 피하고, **광고·구독 수익을 리워드 재원으로 직접 연동**한다.

| 항목 | 내용 |
|------|------|
| **리워드 재원** | 광고 수익의 30% cap |
| **지급 단위** | BTC (사토시, Lightning Network) |
| **보상 철학** | 고정 소득 X → 기여 기반 변동 보상 |

### 리워드 표시 방식 — 스코어 UI (감독자 결정 2026-05-22)

> **법적 이슈 방지를 위해 앱 내 리워드는 "스코어(점수)" 형태로 표시한다. 가상자산 잔액 표시 방식을 사용하지 않는다.**

- 앱 내에서는 포인트·코인 명칭 대신 **스코어(점수)** 로 표현
- 클레임(출금) 플로우:
  1. 사용자가 클레임 요청
  2. 광고 시청 (보상 재원 조달)
  3. 광고 시청 완료 시 사용자가 등록한 **Lightning 주소**로 BTC 즉시 전송

### 기여 기반 리워드 구성 요소 (Phase B 확정 예정)

- 업로드 횟수 (Phase A: 이것만 사용)
- 다른 유저와의 상호작용 (Phase A: 기본 포함)
- 영상 좋아요 수
- 영상 퀄리티 점수 (어드바이저 평가 또는 AI 점수)

> **Phase A 단순화**: 업로드횟수 + 상호작용만으로 산정. 퀄리티 점수는 Phase B.

### 규제 대응

한국 가상자산이용자보호법(2024 시행), 2027년 과세 개시 예정.  
스코어 UI + 광고 시청 후 클레임 방식으로 법적 가상자산 서비스 해당 여부 최소화.  
VASP 등록 의무 해당 여부는 Phase B 시작 전 법무 확인 필수.

## 이벤트 & 챌린지 (Phase B)

- **월 단위 챌린지**: 예) 5월 달리기 챌린지 — 30일간 기록 공유
- **루틴 공유**: 어드바이저가 설계한 4주 루틴을 팔로워가 따라 기록
- 완료 증빙 기준과 어뷰징 탐지 로직은 Phase B 설계 전 확정
