## R1 의견

### 기획
**핵심 주장**: 비전 재정의 방향은 옳지만 2026-05-28 MVP 런칭 7일 전 범위 폭발을 경계해야 한다.
- "운동 성장 일기" 정체성은 광고 기반 수익 모델보다 커뮤니티 체류율 + 기여 리워드와 정합성 높음
- 5가지 질문, 음성 합성, 역할 구분은 각각 신규 개발 범위
- 기여 기반 리워드는 비트코인 지급 산정 복잡화 → 인허가 리스크 심화
- **제안**: vision.md에 Phase A/B 분리 테이블 필수 삽입

### 개발
**핵심 주장**: 영상+음성 자동 병합이 가장 복잡한 차별점이며 v0.2로 분리 필요.
- 현재 스택은 피드+리워드까지 커버 가능, 음성 병합은 FFmpeg 서버 부하 문제
- 5-28 런칭 목표 6일 남은 상황에서 음성 병합+챌린지+역할 분기 동시 구현 불가
- **제안**: 클라이언트 사이드 MediaRecorder API로 단일 파일 업로드, 서버 부하 없음

### 디자인
**핵심 주장**: 5가지 업로드 질문이 핵심 UX 접점이다.
- 텍스트 레이어(종목·강도·개선점) 피드 노출로 정체성 차별화
- 업로드 질문 5개 → 스텝 카드 UI(1/5→5/5)로 완료 진행감 부여
- **우려**: 마찰로 작용하면 D1 이탈률 급증

### QA
**핵심 주장**: 기여 기반 리워드는 어뷰징 탐지 없이 지급하면 즉시 봇 어뷰징 발생.
- 어드바이저/운동인 역할 이중화는 6일 내 불가
- **제안**: 리워드 산정 공식과 어뷰징 판단 임계값 수치 명시 필수

### 악마의 변호인
**핵심 주장**: "커뮤니티가 본질"이라는 재정의는 리워드 없이 초기 유입 불가능하다는 사실을 회피한다.
- 질문 5개 + BTC 지갑 보유 전제 → 신규 유저 온보딩 이탈률 치명적
- 기여 기반 리워드 → 초기 100명 규모에서 파워유저 독점 구조 형성
- 음성+영상 병합은 기능이지 차별점이 아님 (CapCut·Shorts가 이미 무료 제공)
- **도전 질문**: "커뮤니티가 본질이라면 BTC 없어도 가입할 이유가 있는가?"

### 재무
**핵심 주장**: 기여 기반 리워드는 측정 단위 없이 구현 불가, 리워드 재원 cap 설정 필수.
- 광고 수익 30% cap → 월 30만원 풀 → 100명 기준 인당 3,000원/월 BTC
- **제안**: 어드바이저 수익화 모델 방향 명기 필요

### 마케팅
**핵심 주장**: "비트코인 후크 + 운동 성장 일기"는 강력한 조합이나 타깃 분산 위험.
- Sweatcoin 사례: 보상이 입문 동기, 커뮤니티가 잔존율 결정 → 방향 맞음
- **제안**: 달리기 단일 카테고리로 초기 집중 후 확장, "한입 비트코인" 브랜드 연계

### 리서처
**핵심 주장**: Move-to-earn 시장 $100억 성장세, 단 StepN식 폰지 탈피 필수.
- Move-to-earn 앱 시장 2024년 $25억 → 2033년 $100억 (CAGR 17.4%)
- StepN 붕괴: 신규 유입 자본 의존 폰지 + GST 97% 폭락
- 한국 VASP 규제(2024 시행) + 2027년 과세 개시 → 법무 검토 필수

### 운영
**핵심 주장**: 리워드 산정 기준 불투명하면 첫 달 분쟁 발생.
- 하루 3회 업로드 제한 구현으로 기본 남용 방지 존재
- **제안**: 초기 1개월 업로드횟수+상호작용만으로 단순화, 리워드 공식 SOP 문서화

## 합의 및 결정

### 합의된 사항

1. **결정**: 이 앱의 정체성은 "운동 성장 일기(Workout Growth Diary)"다. 비트코인은 진입 후크이며 커뮤니티와 성장 피드백이 핵심 동기다.
   - 지지: 기획, 마케팅, 리서처, 운영, 재무 (5개 역할)

2. **결정**: Phase A(~2026-05-28)는 영상 업로드 + 피드 + 기본 상호작용 + 어뷰징 방어(해시 중복 검증)만 포함한다. 업로드 전 5가지 질문, 어드바이저 역할, 챌린지·루틴, 음성 병합은 모두 Phase B 이후.
   - 지지: 기획, 개발, 디자인, QA, 운영 (5개 역할)

3. **결정**: 리워드 재원 = 광고 수익의 30% cap, StepN식 폰지 구조를 명시적으로 배제한다.
   - 지지: 재무, 리서처, 마케팅, 기획 (4개 역할)

4. **결정**: 초기 리워드 산정은 업로드횟수 + 상호작용만으로 단순화한다 (퀄리티 점수는 Phase B).
   - 지지: 운영, QA, 개발, 기획 (4개 역할)

5. **결정**: `docs/vision.md` 작성 완료. 모든 에이전트는 구현 전 이 문서를 읽는다.
   - 지지: 전원

### 미해결 쟁점

1. **미해결**: 초기 집중 운동 카테고리 — 마케팅(달리기) vs 리서처(홈트/맨몸운동)
   - 달리기: Strava 브랜드 연계 각도 확보, 단 레드오션
   - 홈트: 진입장벽 낮고 업로드 UX와 자연스러움, 단 비트코인 연계 스토리 약함

2. **미해결**: 인당 3,000원/월 BTC가 실제 동기 부여 임계값을 충족하는가
   - 재무: BTC 보유 경험이 동기 (미검증 가정)
   - 악마: 데이터 없이 확정 불가

3. **미해결**: 볼륨 정규화(Web Audio API) Phase A 포함 여부
   - 디자인: 최소 포함 필요 (시청 경험)
   - QA/개발: 테스트 리스크 존재
