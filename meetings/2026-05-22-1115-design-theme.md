## R1 의견

### 기획
테마 전환 기능은 Phase A 범위 밖. 5/28 런칭 목표 기준 1개 테마 빠르게 확정이 맞다. 테마 선정 기준 핵심: 비트코인/운동 브랜드와 시각적 연결고리가 명확할 것.

### 개발
CSS 변수 방식이 최적. `data-theme` attribute를 `<html>`에 토글 + `index.css` 테마 변수 선언으로 런타임 전환 가능. 초기 추정 2~3시간.

### 디자인
오렌지는 Sweatcoin과 겹치고, 다크 배경에서 눈 피로 유발, WCAG 대비 부적합 구간 多. 5가지 테마 제안:
1. **Bitcoin Black** — `#F7931A` 액센트 + `#0A0A0A` 배경 (다크 금융 프리미엄)
2. **Neon Volt** — `#00FF87` 네온그린 + `#0D0D0D` 배경 (사이버펑크 에너지)
3. **Arctic Gold** — `#FFD700` 순금 + `#0A1628` 딥네이비 (럭셔리 스포츠)
4. **Midnight Steel** — `#6C63FF` 인디고 + `#0F0F0F` 배경 (테크+크리에이터)
5. **Pure White Sport** — `#FFFFFF` 라이트 + `#F7931A` 포인트 소량 (애플 피트니스 레퍼런스)

추천: **Bitcoin Black** — 피드 영상 몰입, BTC 골드 계층화, 리워드 숫자 "금액처럼" 노출.

### QA
주요 실패 시나리오: (1) Zustand 구독 누락 → mixed theme 렌더링, (2) 영상 플레이어 iframe 내 컨트롤 contrast 붕괴. 5테마 × core flow E2E 매트릭스 테스트 선행 필요.

### 악마의 변호인
① 런칭 5일 전 전체 컴포넌트 수정 → 회귀 리스크. ② 사용자 0명 상태에서 테마 수요 미검증. ③ 기회비용 — 어뷰징 감지·지급 안정성 > UI 폴리싱. ④ 오렌지 제거 시 비트코인 연상 약화 가능.

### 재무
MAU 0 상태에서 테마 ROI 실질 제로. 광고 CPM은 MAU·세션 수에 종속. 개발 시간 1~2주 투자 대비 수익화 효과 없음.

### 마케팅
오렌지는 Sweatcoin이 선점, 저가 이미지 고착. 다크 배경+골드 액센트가 "부의 증식" 연상 즉시 구현. 비트코인 커뮤니티가 초기 100명 확보에 더 유리한 타겟.

### 리서처
피트니스 앱(Nike/Apple Fitness+) + 크립토 앱 모두 다크 모드로 수렴. M2E 시장(2024 $669.9M, CAGR 17.7%)에서 다크+네온그린+골드 조합이 두 카테고리 교차점. 테마 전환 기능이 2025년 앱 표준화 추세.

### 운영
테마 전환 = QA 케이스 5배 증가. 런칭 전 1개 확정이 운영 부담 최소화. 사용자 피드백은 Telegram 봇/설문으로 충분.

## R2 토론

### 기획
개발 "2~3시간"은 신규 컴포넌트 기준이며 기존 리팩토링 비용 미포함 → 실제 비용은 1~2일. 리서처 "시장 표준화" 주장은 MAU 0에서 성립 안 함. **결론: Bitcoin Black 1개 확정, 테마 전환 기능은 Phase B 백로그로 명시 이월.**

### 개발
공수 재산정: CSS 변수 1h + Zustand 동기화 1.5h + 기존 컴포넌트 하드코딩 교체 **3~5h** + 시각 회귀 테스트 1h = **총 6.5~8.5h**. QA E2E 매트릭스는 CSS 변수 방식이면 1.5배 수준으로 과장. 1테마 고정 시 추가 공수 제로 → 5/28 데드라인엔 1테마가 유일한 안전 선택.

### 디자인
악마 반론(오렌지 제거 → 연상 약화) 조건부 인정. 단, #F7931A를 배경이 아닌 액센트로 격상하면 연상 유지 가능. **절충안: 5개 테마 CSS 블록만 tokens.css에 작성, 감독자가 DevTools로 비교 후 1개 확정. 전환 UI 컴포넌트 없음 → E2E 불필요.**

### QA
테마 교체(1→1): 기존 suite 1회 재실행, 0.5h. 테마 전환(1→5): 상태 조합 폭발, 3~5h. **1개 교체 방식이면 QA 부담 90% 절감. 5테마 전환 UI는 5/28과 양립 불가.**

### 악마의 변호인
공수 과소평가 경고 유지. Bitcoin Black의 #F7931A = 기존 오렌지와 동일 코드 → "골드 재명명" 가능성 지적. 실제 하드코딩 색상 범위 grep 없이 공수 추정 수용한 것은 미검증 가정.

### 재무
마케팅 "다크+골드 → 100명 가속" ROI 계산: MAU 100명 × RPM $1 = 월 광고 수익 수백원 수준. 테마 선택의 재무 영향은 MAU 1,000명 이상부터 측정 가능. 재무 변수로 테마 선택 불가.

### 마케팅
악마 반론 논파: #F7931A는 다크 배경 위에서 더 선명히 돌출 → 비트코인 연상 약화가 아닌 강화. 악마의 전제("오렌지 제거")가 틀렸다. 액센트 강화 = 마케팅 관점 긍정.

### 리서처
자기 비판 수용: "테마 전환 = 시장 표준" 주장은 대형 앱 기준, MVP 100명 단계에 과잉 적용. 초기 사용자 획득은 테마가 아닌 보상 신뢰성·바이럴 UX에 종속. **R1 주장 철회, 1개 테마 확정 지지.**

### 운영
/admin 숨김 UI 절충안 조건부 수용: (1) /admin 인증 필수, (2) 테마 변경 시 Cloudflare 캐시 퍼지 SOP 필요. 이 조건 충족 시 일반 사용자 QA 케이스 증가 없음.

## 합의 및 결정

### 합의된 사항

1. **Bitcoin Black 단일 테마로 Phase A 확정** `결정`  
   배경 `#0D0D0D~#1A1A1A`, 강조 `#F7931A` (BTC 공식 골드), 다크 UI 기반.  
   오렌지를 배경색에서 액센트로 격상 — 비트코인 연상 유지하면서 시각 계층 분리.  
   *(기획·디자인·마케팅·리서처·개발 5개 역할 지지)*

2. **테마 전환 UI 기능은 Phase B로 명시 이월** `결정`  
   QA 케이스 폭발, 개발 공수 6.5~8.5h, ROI 미검증 이유로 런칭 전 제외.  
   *(기획·재무·운영·QA·리서처 5개 역할 동의)*

3. **5개 테마 CSS 변수 블록만 tokens.css에 작성** `결정`  
   감독자가 DevTools로 비교 후 1개 확정. 전환 UI 없음 → E2E 매트릭스 불필요.  
   *(디자인 제안, 개발 동의)*

### 미해결 쟁점

- **"Bitcoin Black = 오렌지 재포장" 논란**: `#F7931A` 코드 동일 사용. 마케팅은 "다크 배경에서 더 선명"으로 반박, 악마는 "명칭 변경"이라 주장. 감독자 실제 렌더링 확인 필요.
- **개발 공수 범위**: 기존 컴포넌트 하드코딩 색상 grep 조사 선행 없이 일정 확정 불가.

### 감독자 결정 필요

1. 🔴 **즉시**: `tokens.css` 5개 테마 블록 구현 → DevTools 또는 앱 내에서 직접 확인 후 최종 테마 1개 선택
2. 🟡 **이번 주**: /admin 테마 프리뷰 UI 추가 여부 (운영 조건부 수용 vs 기획 Phase B 권고)

### ⚠️ 악마의 변호인 미해소 경고

Bitcoin Black의 `#F7931A`는 현재 오렌지와 동일한 hex 코드다. "다크 배경 위 액센트"라는 설계 차이는 있으나, 감독자가 원한 "오렌지 스타일 제거"가 색상 코드 변경인지 배경 변경인지 재확인 필요. 5개 테마 DevTools 비교 후 최종 판단 권고.
